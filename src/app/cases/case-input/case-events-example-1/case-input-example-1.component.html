Prop drilling with reference vs mutation

All messages from senders are emitted to Root, then passed down to target Node - prop drilling.

Observe how you can't directly update Node-2 message. It is going to be updated only if other message is passing to Node-2-1 or Node-2-2. This is happening as one of @input reference changes in Node-2 due to prop drilling.

Reload example couple of times and try different scenarios.

  1. Emits message and mutates data object of Node-2
  2. Emits message and changes data reference of Node-2-1
  3. Emits message and changes data reference of Node-2-2
