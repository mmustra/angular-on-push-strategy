Service communication and @Input

All messages from senders are emitted to parent Node via service, which then passes it via @Input to child node. Some receivers have their messages initialized with setTimeout().

Service communication via observables don't trigger change detector. Only the part where Node is passing message by changing @Input reference does.

Reload example couple of times and try different scenarios.

  1. Emits message via service to Node-2-2, which passes it via @Input to Node-2-2-1
  2. Emits message via service to Node-2-1, which passes it via @Input to Node-2-1-1
